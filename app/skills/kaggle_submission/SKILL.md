# Kaggle Submission Skill

## Purpose

Prepare CareerVerse Agent for Kaggle Capstone submission.

## Track

Agents for Good

## Submission Requirements

The project should support:

- Kaggle Writeup
- Media Gallery assets
- YouTube demo video
- Public project link
- Public GitHub repository
- Clear README
- Setup instructions
- Architecture explanation
- Demo-ready API

## Course Concepts to Demonstrate

At least three concepts should be visible in code or video:

- Agent / Multi-agent system
- MCP-style tool integration
- Agent Skills
- Security features
- Deployability
- Local evaluation pipeline

## Writeup Rules

The writeup must:

- Stay under 2,500 words.
- Explain the problem clearly.
- Explain why agents are useful.
- Show architecture.
- Explain implementation.
- Mention security and evaluation.
- Separate implemented features from future work.
- Avoid claiming unimplemented features.

## Demo Video Structure

Recommended 5-minute flow:

```text
0:00-0:20 Problem
0:20-0:50 Solution
0:50-1:40 Architecture
1:40-3:20 Demo
3:20-4:20 Course concepts
4:20-5:00 Conclusion
```

## Final Checklist

Before submission:

* Repo is public.
* README is complete.
* No secrets are committed.
* Demo works.
* Video is attached.
* Project link is attached.
* Writeup is submitted, not left as draft.
